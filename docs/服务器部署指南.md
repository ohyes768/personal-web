# 服务器部署指南

本文档说明如何在阿里云 ECS 上部署 personal-web 项目。

## 前提条件

1. douying-collect 服务已在服务器上运行
2. 已有 douying-collect:latest 镜像

## 部署步骤

### 1. 准备工作

```bash
# 上传代码到服务器
cd /root
git clone https://github.com/ohyes768/personal-web.git
cd personal-web

# 或者使用 scp 上传
# scp -r ./personal-web root@your-server-ip:/root/
```

### 2. 检查 douying-collect 镜像

```bash
# 查看本地镜像
docker images | grep douying-collect

# 如果镜像不存在，需要先构建
cd /root/douying-collect
docker build -t douying-collect:latest .
```

### 3. 配置环境变量

```bash
# 复制环境变量配置
cd /root/personal-web
cp .env.example gateway/.env

# 编辑配置（根据需要修改）
vim gateway/.env
```

**gateway/.env 关键配置**:
```bash
# Docker 网络中使用服务名访问
DOUYIN_SERVICE_URL=http://douying-collect:8093
```

### 4. 检查 Docker 网络

```bash
# 如果 douying-collect 已在运行，查看其网络
docker inspect douying-collect | grep -A 10 "Networks"

# 记录网络名称，例如：bridge 或自定义网络
```

如果 douying-collect 在默认的 bridge 网络，docker-compose.yml 会自动处理。
如果使用自定义网络，需要修改 docker-compose.yml：

```yaml
networks:
  web-network:
    external: true
    name: <douying-collect的网络名称>
```

### 5. 启动服务

```bash
# 方式1: 使用部署脚本（推荐）
./scripts/deploy.sh

# 方式2: 手动启动
docker compose -f docker-compose.yml up -d
```

### 6. 验证部署

```bash
# 检查容器状态
docker ps | grep -E "api-gateway|personal-web|douying-collect"

# 检查服务健康
curl http://localhost:8080/api/health    # Gateway
curl http://localhost:8093/api/health    # Douying-Collect
curl http://localhost:3000               # Frontend

# 查看 Gateway 日志
docker logs api-gateway -f

# 测试 Gateway 到 douying-collect 的连通性
docker exec api-gateway curl http://douying-collect:8093/api/health
```

## 常见问题

### Q1: 启动失败，提示找不到网络

**解决方法**:

```bash
# 创建外部网络
docker network create web-network

# 或者修改 docker-compose.yml，移除 external: true
```

### Q2: Gateway 无法访问 douying-collect

**解决方法**:

```bash
# 1. 确认两个容器在同一网络
docker network inspect <network-name> | grep -A 5 "Containers"

# 2. 如果不在同一网络，将 gateway 加入 douying-collect 的网络
docker network connect <douying-collect的网络名> api-gateway

# 3. 或者重启服务，让 docker-compose 管理
docker compose -f docker-compose.yml down
docker compose -f docker-compose.yml up -d
```

### Q3: douying-collect 镜像不存在

**解决方法**:

```bash
# 方式1: 从 Docker Hub 拉取（如果已推送）
docker pull douying-collect:latest

# 方式2: 在服务器上重新构建
cd /root/douying-collect
docker build -t douying-collect:latest .

# 方式3: 从本地导出镜像并导入
# 在本地机器上：
# docker save douying-collect:latest -o douying-collect.tar
# 
# 上传到服务器后：
# docker load -i douying-collect.tar
```

### Q4: 端口冲突

**解决方法**:

```bash
# 检查端口占用
netstat -tulpn | grep -E "8080|8093|3000"

# 如果端口被占用，修改 docker-compose.yml 中的端口映射
ports:
  - "8081:8080"  # 将宿主机端口改为 8081
```

## 服务管理

### 启动服务

```bash
# 启动所有服务
./scripts/deploy.sh

# 仅启动特定服务
docker compose -f docker-compose.yml up -d gateway
docker compose -f docker-compose.yml up -d frontend
```

### 停止服务

```bash
# 停止所有服务
./scripts/stop-prod.sh

# 仅停止特定服务
docker compose -f docker-compose.yml stop gateway
```

### 查看日志

```bash
# 查看所有服务日志
docker compose -f docker-compose.yml logs -f

# 查看特定服务日志
docker logs api-gateway -f
docker logs douying-collect -f
docker logs personal-web-frontend -f
```

### 重启服务

```bash
# 重启所有服务
./scripts/deploy.sh restart

# 重启特定服务
docker restart api-gateway
```

## 更新部署

### 更新代码

```bash
cd /root/personal-web

# 拉取最新代码
git pull

# 重新构建镜像（如果需要）
docker compose -f docker-compose.yml build gateway
docker compose -f docker-compose.yml build frontend

# 重启服务
docker compose -f docker-compose.yml up -d
```

### 仅更新配置

```bash
# 修改环境变量后
vim gateway/.env

# 重启相关服务
docker restart api-gateway
```

## 安全建议

1. **配置防火墙**: 只开放必要的端口（80, 443, 22）
2. **使用 HTTPS**: 配置 Nginx SSL 证书
3. **定期更新**: 定期更新系统和 Docker 镜像
4. **监控日志**: 定期检查服务日志
5. **数据备份**: 定期备份重要数据

## 外部访问配置

### 使用 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    location /api/ {
        proxy_pass http://localhost:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 配置 SSL（Let's Encrypt）

```bash
# 安装 certbot
apt install certbot python3-certbot-nginx

# 获取证书
certbot --nginx -d your-domain.com

# 自动续期
certbot renew --dry-run
```
