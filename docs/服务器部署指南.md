# NAS 服务器部署指南

本文档介绍如何在 NAS 服务器上部署 personal-web 项目。

## 前置条件

### 1. 服务器环境
- **操作系统**: Ubuntu 20.04+ / Debian 10+
- **Docker**: 已安装 Docker 和 Docker Compose v2
- **Nginx**: NAS 上已有 Nginx 反向代理（已在端口 80/443 监听）

### 2. Git Submodule 说明

项目包含 **douyin-processor** 作为 Git Submodule：
- **子模块路径**: `backend/douyin-processor`
- **子模块仓库**: https://github.com/ohyes768/douyin-processor.git
- **克隆要求**: 必须使用 `--recurse-submodules` 参数

## 部署步骤

### 1. 准备代码

```bash
# 克隆仓库（包含 submodule）
git clone --recurse-submodules https://github.com/ohyes768/personal-web.git
cd personal-web

# 如果已经 clone 但还没有 submodule
git submodule update --init --recursive
```

### 2. 配置环境变量

在项目根目录创建 `.env` 文件：

```bash
cat > .env << 'EOF'
DIVIDEND_ALIYUN_ACCESS_KEY=你的阿里云key
DOUYIN_ALIYUN_ACCESS_KEY=你的阿里云key
EOF
```

### 3. 启动服务

```bash
# 构建并启动
docker compose -f docker-compose.nas.yml up -d --build

# 查看服务状态
docker compose -f docker-compose.nas.yml ps
```

### 4. 验证

```bash
# 检查 dividend 后端健康
curl http://localhost:8092/api/dividend/health

# 检查 dividend 前端
curl http://localhost:3003/dividend
```

## Nginx 反向代理配置

在 NAS 上的 Nginx 中添加以下路由规则：

```nginx
# 股息率
location /dividend {
    proxy_pass http://127.0.0.1:3003;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_set_header Host $host;
}

location /api/dividend {
    proxy_pass http://127.0.0.1:8092;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_connect_timeout 300s;
    proxy_send_timeout 300s;
    proxy_read_timeout 300s;
}

# 抖音
location /douyin {
    proxy_pass http://127.0.0.1:3004;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_set_header Host $host;
}

location /api/douyin {
    proxy_pass http://127.0.0.1:8093;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}
```

## 常用命令

### 启动/停止
```bash
docker compose -f docker-compose.nas.yml up -d
docker compose -f docker-compose.nas.yml down
```

### 重启
```bash
docker compose -f docker-compose.nas.yml restart
```

### 更新部署
```bash
git pull
git submodule update --init --recursive
docker compose -f docker-compose.nas.yml build --no-cache
docker compose -f docker-compose.nas.yml up -d
```

### 查看日志
```bash
docker compose -f docker-compose.nas.yml logs -f
docker compose -f docker-compose.nas.yml logs -f dividend-backend
```

### 进入容器
```bash
docker exec -it dividend-backend bash
docker exec -it dividend-frontend sh
```

## 数据卷

| 卷名 | 用途 |
|------|------|
| dividend-data | 股息率数据 |
| dividend-logs | 股息率服务日志 |
| dividend-config | 股息率配置 |
| douyin-data | 抖音视频数据 |
| douyin-logs | 抖音服务日志 |

## 故障排查

### 1. 服务无法启动
```bash
docker compose -f docker-compose.nas.yml logs
```

### 2. 健康检查失败
```bash
curl http://dividend-backend:8092/api/dividend/health
```

### 3. 网络问题
确认 Docker 网络 `nginx_nginx-bridge` 已存在：
```bash
docker network ls | grep nginx
```
