## 项目结构

```
personal-web/
├── apps/                     # 5 个独立 Next.js 前端
│   ├── dividend/             # A 股高股息分析（端口 3003）
│   ├── douyin/               # 抖音视频转写（端口 3004）
│   ├── economic/             # 宏观经济（端口 3001）
│   ├── fund-flow/            # 资金流向（端口 3002）
│   └── news/                 # 新闻联播分析（端口 3005）
├── backend/                  # 3 个 Python 后端（git submodule）
│   ├── dividend-select/      # 股息率后端（FastAPI，端口 8092）
│   ├── douyin-processor/     # 抖音后端（端口 8093）
│   └── global-macro-fin/     # 宏观金融后端（端口 8094）
├── nginx/                    # 示例 Nginx 配置
├── scripts/                  # Windows 开发脚本
├── docs/                     # 文档目录
└── docker-compose.yml        # 本地开发 Docker Compose
└── docker-compose.nas.yml    # NAS 生产 Docker Compose
```

## Git Submodule

| 子模块 | 仓库 | 说明 |
|--------|------|------|
| `backend/dividend-select` | ohyes768/dividend-select | A 股股息率后端 |
| `backend/douyin-processor` | ohyes768/douyin-processor | 抖音视频转写后端 |
| `backend/global-macro-fin` | ohyes768/global-macro-fin | 宏观金融后端 |

克隆时需使用 `git clone --recurse-submodules <url>`。

## 故障排查

### Q1: 如何在 NAS 上部署？
**A**: 参考 [docs/deployment.md](deployment.md) 或 [DOCKER_DEPLOY.md](../DOCKER_DEPLOY.md)。

### Q2: 如何查看日志？
**A**:
```bash
# 本地
docker compose logs -f

# NAS
docker compose -f docker-compose.nas.yml logs -f dividend-backend
```

### Q3: 如何备份数据？
**A**: 参考 [docs/deployment.md](deployment.md) 中的数据备份章节。

## 维护指南

1. 更新子模块代码 → 在子模块内 commit + push
2. 回到主仓库 → `git add backend/<submodule>` 更新 gitlink
3. 主仓库 commit + push
4. 在 NAS 上 `git pull && git submodule update --init --recursive`
