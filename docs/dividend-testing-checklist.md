# 股息率 Web 测试点清单

> 后续开发或重构股息率相关功能时，必须按本清单完成检查。未通过的项目需要先修复，不能只说明“看起来没问题”。

## 一、每次改动都必须检查

### 1. 代码能正常构建

- 前端股息率页面必须能完成生产构建。
- 后端测试必须能被正常发现并运行，不能在收集阶段就失败。
- 如果因为本机工具缺失、网络限制、依赖缺失导致不能运行，必须明确说明原因，并补充已完成的替代检查。

建议命令：

```powershell
cd F:\github\person_project\personal-web\apps\dividend
.\node_modules\.bin\next.CMD build

cd F:\github\person_project\personal-web\backend\dividend-select
python -m pytest tests
```

### 2. 页面能打开

- 启动后端服务。
- 启动股息率前端页面。
- 打开页面后不能白屏。
- 页面标题、返回入口、筛选区、数据表格、更新按钮都应正常显示。
- 浏览器控制台不能出现会影响页面使用的错误。

建议人工访问：

- 前端：`http://localhost:3003`
- 后端健康检查：`http://localhost:8092/api/dividend/health`

### 3. 基础数据能加载

- 页面首次进入后能请求股票列表。
- 有数据时表格应显示股票代码、名称、交易所、股息率等核心字段。
- 无数据时应显示清晰的空状态，而不是白屏或一直加载。
- 后端返回错误时，页面应显示错误提示。

### 4. 仓库不能被运行产物污染

检查本次改动是否误加入以下内容：

- `.next`
- `node_modules`
- `.venv`
- `__pycache__`
- `.pytest_cache`
- `logs`
- 临时 CSV 导出文件
- 临时调试文件
- 本机路径相关文件

建议命令：

```powershell
cd F:\github\person_project\personal-web
git status --short
```

## 二、改了前端页面时追加检查

### 1. 筛选和查询

- 默认进入页面时按默认股息率阈值加载。
- 修改股息率阈值后点击查询，表格结果会刷新。
- 修改交易所后点击查询，表格结果会刷新。
- 输入空值、0、小数时页面不能崩溃。

### 2. 表格展示

- 股票代码、股票名称、交易所、行业、股息率、M120、实时价格等列不应错位。
- 数字为空时应显示统一占位，不应显示 `NaN`、`undefined`、`null`。
- 排序、横向滚动、固定表头或虚拟滚动如果存在，必须在数据较多时仍可用。
- 移动端或窄屏下不能出现严重遮挡。

### 3. 详情弹窗

- 点击季度、行业、年度分红、波动等详情入口能打开弹窗。
- 弹窗内容和当前股票一致。
- 图表有数据时能显示。
- 图表无数据时有明确提示。
- 关闭弹窗后页面状态正常。

### 4. 股票对比

- 可以选择股票加入对比。
- 超过最大选择数量时不能继续加入。
- 少于 2 只股票时不能打开有效对比。
- 打开对比抽屉后，字段显示完整。
- 移除股票、清空选择后状态正确。

### 5. 导出 CSV

- 点击导出后能下载 CSV。
- CSV 文件名包含日期。
- CSV 表头和页面字段一致。
- 中文内容打开后不应乱码。
- 空值不应导出成 `undefined` 或 `null`。

## 三、改了数据刷新功能时追加检查

### 1. 股息率数据刷新

- 未刷新时按钮状态正确。
- 点击刷新后按钮进入加载状态。
- 刷新成功后显示成功信息。
- 部分失败时能看到失败数量。
- 刷新完成后页面能重新读到最新状态。
- 重复点击不能触发并发刷新。

### 2. M120 数据刷新

- 状态接口能判断是否需要更新。
- 点击刷新后能完成请求。
- 刷新后表格里的 M120、偏离度、实时价格相关字段能更新。
- 没有 M120 数据时页面不崩溃。

### 3. 实时价格刷新

- 点击刷新实时价格后能完成请求。
- 刷新后页面显示实时价格和偏离度。
- 外部行情接口失败时页面显示错误，不应白屏。

### 4. 财务数据刷新

- 能识别缺失财务数据的股票。
- 点击刷新后能完成请求。
- 刷新成功后缺失数量变化合理。
- 财务指标不能计算时应显示清晰占位。

## 四、改了后端接口时追加检查

### 1. 健康检查

- `/api/dividend/health` 能返回服务状态。
- 数据文件不存在时返回明确状态，不能直接崩溃。

### 2. 股票列表接口

- `/api/dividend/stocks` 能返回列表。
- 支持最小股息率筛选。
- 支持交易所筛选。
- 支持排序。
- 空数据时返回空列表。
- 单条数据字段完整，前端需要的字段不能缺失。

### 3. 股票详情接口

- `/api/dividend/stocks/{code}` 能返回指定股票。
- 不存在的股票代码返回清晰错误。
- 季度数据、年度分红、行业信息字段结构稳定。

### 4. 状态接口

- 股息率状态接口能返回是否需要更新。
- M120 状态接口能返回是否需要更新。
- 财务状态接口能返回缺失数量和缺失股票。

### 5. 刷新接口

- 刷新接口成功时返回处理数量。
- 外部数据源失败时返回清晰错误。
- 已有刷新任务进行中时应阻止重复刷新。
- 刷新过程不能写入本机临时路径，例如 `c:/temp`。

## 五、改了数据文件或计算逻辑时追加检查

### 1. CSV 读取

- 中文列名能被正确识别。
- 文件不存在时返回空状态或明确错误。
- 股票代码能保持 6 位。
- 数字列里的空值、横线、异常字符串不会导致崩溃。

### 2. 股息率计算

- 三年平均股息率计算正确。
- 单年缺失数据时处理合理。
- 分红次数筛选符合预期。
- 主板股票筛选符合预期。

### 3. M120 和实时价格计算

- M120 缺失时不会显示错误数字。
- 实时价格缺失时不会显示错误数字。
- 偏离度计算公式稳定。

### 4. 财务指标计算

- 扣非净利润同比为空时显示占位。
- 三年复合增长率为空时显示占位。
- 异常值不会导致页面或接口崩溃。

## 六、改了部署或启动脚本时追加检查

- 启动脚本显示的端口必须和实际端口一致。
- `.env.local` 里的后端地址必须和实际服务一致。
- Docker 配置中的端口必须和应用启动端口一致。
- 一键启动后，前端和后端都能访问。
- 停止脚本不会误杀无关进程。
