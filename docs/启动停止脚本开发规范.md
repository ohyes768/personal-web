# 启动/停止脚本开发规范

## 一、命名规范

| 场景 | 命名格式 | 示例 |
|------|---------|------|
| 启动脚本 | `start-{场景名}-dev.bat` | `start-macro-dev.bat`, `start-douyin-dev.bat` |
| 停止脚本 | `stop-{场景名}-dev.bat` | `stop-macro-dev.bat`, `stop-douyin-dev.bat` |

---

## 二、职责划分

| 脚本类型 | 职责 | 缓存清理 |
|---------|------|----------|
| **启动脚本** | 启动服务、环境准备 | ✅ **必须**：启动前清理缓存 |
| **停止脚本** | 停止服务、关闭窗口 | ❌ **禁止**：不负责缓存清理 |

**设计理念**: 缓存清理应该在启动时进行，确保每次启动都使用最新代码。

---

## 三、启动脚本开发要求

### 1. 基础结构

```batch
@echo off
setlocal enabledelayedexpansion

REM 打印标题
echo ========================================
echo   Starting {场景名} Dev Environment
echo ========================================

REM 执行流程
echo [1/N] 启动服务A...
REM 清理缓存
REM 检查环境
REM 启动服务

echo 等待 X 秒...
timeout /t X /nobreak

echo [2/N] 启动服务B...
...

REM 显示服务地址
echo ========================================
echo   Dev Environment Started
echo ========================================
echo Service A:  http://localhost:{portA}
echo Service B:  http://localhost:{portB}

pause
```

### 2. 必须包含的功能

#### (1) 缓存清理（启动前）
```batch
echo Cleaning Python cache...
if exist "src\__pycache__" rmdir /s /q "src\__pycache__" 2>nul
if exist "src\api\__pycache__" rmdir /s /q "src\api\__pycache__" 2>nul
if exist "src\services\__pycache__" rmdir /s /q "src\services\__pycache__" 2>nul
if exist "src\utils\__pycache__" rmdir /s /q "src\utils\__pycache__" 2>nul
echo Cache cleared.
```

#### (2) 虚拟环境检查
```batch
if not exist ".venv" (
    echo Creating virtual environment...
    python -m venv .venv
    .venv\Scripts\python -m pip install -r requirements.txt
)
```

#### (3) 依赖检查
```batch
if not exist "node_modules" (
    echo Installing dependencies...
    npm install
)
```

#### (4) 独立窗口启动
```batch
REM 后端服务
start "{服务名}" cmd /k ".venv\Scripts\activate && python -m uvicorn src.main:app --reload --host 0.0.0.0 --port {port}"

REM 前端服务
start "{服务名}" cmd /k "npm run dev"
```

### 3. 服务启动参数

| 参数 | 说明 |
|------|------|
| `--reload` | 代码变更自动重载（开发模式） |
| `--host 0.0.0.0` | 监听所有网络接口 |
| `--port {port}` | 指定端口 |

## 四、停止脚本开发要求

### 1. 执行流程

```
[1/3] 按端口停止服务（使用 taskkill /T 终止进程树）
[2/3] 清理孤儿子进程（仅限项目内）
[3/3] 关闭服务窗口（通过命令行模式匹配）
```

### 2. 必须包含的功能

#### (1) 按端口停止服务（进程树）
```batch
for /f "tokens=5" %%a in ('netstat -aon ^| find ":{port} " ^| find "LISTENING"') do (
    set "PID=%%a"
    taskkill /F /T /PID !PID!
)
```

**关键**: 使用 `taskkill /T` 终止整个进程树，避免子进程残留。

#### (2) 调用通用脚本
```batch
powershell -NoProfile -ExecutionPolicy Bypass -File "%~dp0stop-orphan-python.ps1"
powershell -NoProfile -ExecutionPolicy Bypass -File "%~dp0stop-windows.ps1"
```

### 3. 通用脚本调用规范

| 脚本 | 功能 | 参数 |
|------|------|------|
| `stop-orphan-python.ps1` | 清理孤儿子进程 | 自动识别项目根目录 |
| `stop-windows.ps1` | 关闭服务窗口 | 匹配命令行模式 |

## 五、关键技术要点

### 1. 延迟变量展开
```batch
setlocal enabledelayedexpansion

REM 在循环中使用 !PID! 而非 %PID%
for /f "tokens=5" %%a in (...) do (
    set "PID=%%a"
    taskkill /F /T /PID !PID!
)
```

### 2. 获取脚本所在目录
```batch
%~dp0
```

### 3. 保持窗口打开
```batch
start "窗口标题" cmd /k "命令"
```

## 六、孤儿进程清理规范

### 1. 项目路径检查（避免误杀）
```powershell
$projectRoot = Resolve-Path "$PSScriptRoot\.."

if ($cmdLine -like "*$projectRoot*") {
    # 进一步检查是否为孤儿...
}
```

### 2. 孤儿进程判定标准
```powershell
# 双重检查机制
$localLoopback = $conns | Where-Object {
    $_.RemoteAddress -eq "127.0.0.1" -and $_.State -eq "Established"
}

$listening = $conns | Where-Object { $_.State -eq "Listen" }

# 有本地回环 + 不监听端口 = 真正的孤儿
if ($localLoopback -and -not $listening) {
    Stop-Process -Id $proc.Id -Force
}
```

### 3. 窗口匹配模式
```powershell
$patternsMap = @{
    "{服务名A}" = @("端口模式", "命令模式")
    "{服务名B}" = @("npm.*dev", "next.*dev")
}
```

## 七、文件结构规范

```
scripts/
├── start-{场景}-dev.bat        # 启动脚本
├── stop-{场景}-dev.bat         # 停止脚本
├── stop-orphan-python.ps1      # 通用：孤儿进程清理
└── stop-windows.ps1            # 通用：窗口关闭
```

**注意**: `stop-orphan-python.ps1` 和 `stop-windows.ps1` 为通用脚本，无需为每个场景复制。

---

## 八、常见问题修复

| 问题 | 解决方案 |
|------|----------|
| 子进程残留 | 使用 `taskkill /T` 终止进程树 |
| 变量不展开 | 延迟展开 `!变量!` |
| 窗口未关闭 | 变量名避免 `$pid`（与 PowerShell 自动变量冲突） |
| 误杀其他项目 | 添加项目路径检查 |
| 正常服务被杀 | 双重判定：回环 + 不监听 |

## 九、日志规范

启动/停止脚本的日志统一输出到 `logs/` 目录下：

| 脚本 | 日志文件 |
|------|----------|
| `start-{场景}-dev.bat` | `logs/start-{场景}-dev.log` |
| `stop-{场景}-dev.bat` | `logs/stop-{场景}-dev.log` |
