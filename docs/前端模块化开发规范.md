# 前端模块化开发规范

本文档定义各前端应用的独立开发流程。

## 架构概述

项目采用"独立应用"模式：
- **每个前端**（`apps/<name>/`）是独立的 Next.js 项目，有自己的 `package.json`、`pnpm-lock.yaml` 和 `Dockerfile`
- **后端服务**（`backend/<name>/`）通过 git submodule 管理
- **前端不共享代码包**，各应用完全独立

## 目录结构

```
apps/
├── dividend/     # 独立 Next.js 应用，basePath /dividend，端口 3003
├── douyin/        # 独立 Next.js 应用，basePath /douyin，端口 3004
├── economic/      # 独立 Next.js 应用，无 basePath，端口 3001
└── news/          # 独立 Next.js 应用，无 basePath，端口 3005
```

## 开发流程

### 1. 环境准备

每个应用独立安装依赖：

```bash
cd apps/dividend
pnpm install
pnpm dev
```

### 2. 开发

各应用独立开发，不依赖主仓库的其他代码。修改完成后：

```bash
# 在应用目录内
pnpm build        # 生产构建
pnpm lint         # 代码检查
```

### 3. Docker 构建

每个应用的 Dockerfile 结构相同（三阶段 standalone）：

```dockerfile
FROM node:20-alpine AS deps
RUN npm install -g pnpm
COPY package.json pnpm-lock.yaml ./
RUN pnpm install --frozen-lockfile

FROM node:20-alpine AS builder
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN pnpm build

FROM node:20-alpine AS runner
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static
COPY --from=builder --chown=nextjs:nodejs /app/public ./public
CMD ["node", "server.js"]
```

## basePath 配置

| 应用 | basePath | 访问路径 |
|------|---------|---------|
| dividend | `/dividend` | `http://host:3003/dividend` |
| douyin | `/douyin` | `http://host:3004/douyin` |
| economic | 无 | `http://host:3001` |
| news | 无 | `http://host:3005` |

## API 调用

前端通过环境变量指定后端地址：

```bash
# apps/dividend/.env.local
NEXT_PUBLIC_API_BASE_URL=http://localhost:8092
BACKEND_URL=http://localhost:8092/api/dividend
```

生产环境通过 `docker-compose.nas.yml` 中的 `BACKEND_URL` 环境变量配置。

## 独立开发注意事项

1. **不要在 `apps/<app>/` 目录之外创建前端代码**
2. **不要通过 workspace 引用主仓库的其他代码**
3. **每个应用的 node_modules 独立**，不共享
4. **部署时各自构建**，Dockerfile context 为各应用目录
