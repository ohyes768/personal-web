# 实施计划：辅助数据下拉刷新按钮

## Context

### 当前问题
线上部署的股息率页面只有"更新股息率"和"更新财务数据"等 4 个独立按钮。其中：
- **行业信息**（申万行业）完全没有 UI 更新入口，依赖手动维护 `data/个股申万行业映射.csv`
- **户数信息**后端有 status API 但**没有 refresh API**
- 三个辅助数据（行业/财务/户数）都是**季度级数据**，刷新频率低，单独占 3 个按钮浪费界面空间
- 没有任何节流机制，用户可能误点造成 API 配额浪费

### 目标成果
1. 把 3 个低频辅助数据更新合并为**一个下拉按钮**"📥 更新辅助数据 / ✅ 已是最新"
2. 后端**硬阻断 90 天内重复刷新**（force=true 强制刷新，仅调试用）
3. CSV 文件加"数据季度"列（如 `2026Q1`），文件存放在 `data/` 根目录（不放月份子目录）
4. 下拉项显示"上次更新时间 + 距今天数"，过期项打 📥 标、正常的打 ✅ 标

### 参考项目
行业信息更新逻辑可参考 **`F:/github/person_project/sw-info/`**（已存在）：
- 数据源：`pywencai` 库（问财 API），需要 cookie
- 已有脚本：`fixed_extract_industry.py`（拉原始数据）、`stock_level3_mapping.py`（生成个股映射）
- 输出格式与 dividend-select 项目**完全兼容**（`股票代码, 股票简称, 一级行业, 二级行业, 三级行业`）

### 2.2 新建 `src/data/sw_industry_fetcher.py`

**数据源**：`pywencai` 库（问财 API），**而非 akshare**。
参考已有项目 `F:/github/person_project/sw-info/fixed_extract_industry.py`：

```python
import pywencai
import pandas as pd
from typing import Optional

class SwIndustryFetcher:
    def __init__(self, cookie: Optional[str] = None):
        self.cookie = cookie or os.getenv("PYWENCAI_COOKIE", "")
        self.output_file = "个股申万行业映射.csv"

    def fetch_all(self) -> pd.DataFrame:
        """问财 API 拉所有 A 股的 SW 行业分类"""
        df = pywencai.get(
            query="股票代码,股票简称,所属申万行业,总股本,最新价",
            cookie=self.cookie,
            loop=True,       # 翻页
            perpage=100,
            sleep=1,         # 限速
        )
        # 智能重命名（参考 sw-info/fixed_extract_industry.py）
        df = self._smart_rename(df)
        # 解析行业层级 "银行--全国性股份制银行Ⅰ--全国性股份制银行"
        df["一级行业"], df["二级行业"], df["三级行业"] = zip(*df["申万行业"].apply(self._parse_hierarchy))
        return df[["股票代码", "股票简称", "一级行业", "二级行业", "三级行业"]]

    def _parse_hierarchy(self, s: str) -> tuple[str, str, str]:
        parts = str(s).split("--")
        return (parts + ["", "", ""])[:3]

    def _smart_rename(self, df): ...  # 复用 sw-info 的重命名逻辑
```

**关键依赖**：
- 在 `backend/dividend-select/requirements.txt` 加 `pywencai`
- `PYWENCAI_COOKIE` 通过环境变量传入（**需要同花顺登录态 cookie**）
- 没有 cookie 时降级报错：返回 `{"success": false, "message": "未配置 pywencai cookie"}`

```tsx
<div className="relative" onMouseLeave={() => setAuxOpen(false)}>
  <button onClick={() => setAuxOpen(!auxOpen)} disabled={!anyNeedsUpdate || anyLoading}
    className={`... ${!anyNeedsUpdate ? '灰' : 'indigo-600'}`}>
    {anyNeedsUpdate ? <>📥 更新辅助数据</> : <>✅ 已是最新</>}
    <svg>↓</svg>
  </button>
  {auxOpen && (
    <div className="absolute right-0 mt-1 bg-white border rounded shadow-lg z-50 min-w-[260px]">
      {/* 三项菜单：sw_industry / financial / shareholder */}
      {auxItems.map(item => (
        <button disabled={!status?.needs_update || loading}
          onClick={async () => { setAuxOpen(false); await update(item.key); }}>
          <div>{loading ? spinner : (needsUpdate ? '📥' : '✅')} {item.label}</div>
          <div className="text-xs">
            {item.sub} · 上次：{status.last_updated} ({days_since_update}天前)
          </div>
        </button>
      ))}
      {/* 强制刷新开关 */}
      <label className="...">
        <input type="checkbox" checked={auxForce} onChange={...} /> 强制刷新（忽略 90 天节流）
      </label>
    </div>
  )}
</div>
```

按钮 disabled 逻辑：`auxStatuses.X.needs_update` 任何一个为 `true` 时**可点击**（点击展开菜单选择具体哪一项）；`anyLoading` 时禁用。

## 七、关键风险

1. **pywencai cookie 依赖**：问财 API 需要登录态 cookie。生产环境需要：
   - 在部署机器上手动登录同花顺问财 → 抓 cookie
   - 写入 `PYWENCAI_COOKIE` 环境变量
   - 没配置时 `/sw-industry/refresh` 返回明确错误信息
2. **子模块工作流**：`backend/dividend-select` 改动需在子模块内 commit+push，再回主仓库 bump gitlink（顺序不能错）
3. **mojibake 检查**：提交前 `git show` 检查新文件无 BOM/CRLF
4. **首次部署**：3 个新 CSV 文件不存在，刷新会全成功；后端 90 天节流**不阻断**首次刷新（`days is None` 不算 < 90）
5. **强制刷新开关**仅在菜单内可见，默认不勾选，避免误触
6. **pywencai 翻页 1 秒延迟**：全市场 ~5500 只股，预计 60-90 秒完成fetch，比 akshare 慢很多
